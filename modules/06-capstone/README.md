## Introduction/Scenario

**The Final Challenge:** You've learned the fundamentals, explored codebases, built features, reviewed for security, and discussed team practices. Now it's time to have some fun and build something creative.

**Your Mission:** Build a "Wild & Wonderful" AI-powered feature. It can be silly, it can be interesting, it can be anything you want. The only rule is that it has to be a complete, working feature.

## Inspiration Ideas

- **"Passive Aggressive Commit Message Generator"** - Enter what you did, get a commit message dripping with sarcasm
- **"Code Roast Bot"** - Upload your code, get destroyed by an AI comedian
- **"Meeting Could've Been An Email Detector"** - Paste meeting notes, find out if it was actually necessary
- **"Jargon Bullshit Meter"** - Scores corporate buzzword nonsense levels
- **"CSS Speedrun"** - Race to recreate a design. Leaderboard included.
- **"Regex Golf"** - Shortest regex wins. Lose friendships.
- **"AI That Argues With You"** - Takes the opposite position on everything
- **"The Most Overengineered Todo App"** - Microservices, event sourcing, CQRS. For a todo list.
- **"Git Blame Poetry Slam"** - Your git history as dramatic spoken word
- **"Worst Possible Implementation"** - Solves problems in the most inefficient way possible

**Or build literally anything else. Make it weird.**

## Key Concepts

- **Working Approach:** Choose a strategy (Feature-First, Layer-First, Critical Path).
- **Using AI Effectively:** Generate code, but you make the decisions.
- **Evaluation Criteria:** Functionality, Security, Code Quality, Testing, Documentation.

## Success Criteria

- [ ] You have built a working, secure, and maintainable feature.
- [ ] You can demo your feature and explain your technical decisions.
- [ ] You can identify what worked well with AI and what didn't work so well.
- [ ] You know what you would add or improve if you had more time.

## Common Pitfalls

- **"I'll perfect one part first"**
- **"Security later"**
- **"AI will handle complexity"**
- **"Tests at the end"**

## Evaluation Criteria

- **Functionality (30%):** Does it work as expected?
- **Security (25%):** Did you run the security checklist and address any issues?
- **Testing (20%):** Does the feature have a reasonable level of test coverage?
- **Documentation (15%):** Is the feature well-documented?
- **Creativity & Fun (10%):** How creative, fun, or interesting is the feature?
